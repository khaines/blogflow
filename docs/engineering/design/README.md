# Design Documents

Design documents for BlogFlow live in this directory. Each document captures the architecture, security model, performance targets, and rollout plan for a component or significant change.

## Conventions

### Template

All design documents **must** use the template at [`000-template.md`](./000-template.md). It defines the required sections (§1–§10) and provides guidance in HTML comments within each section.

### File Naming

Use **kebab-case** for all design document filenames:

```
overlay-filesystem.md
configuration-system.md
webhook-handler.md
content-pipeline.md
```

### Organization

Category subdirectories can be created when the number of documents warrants it:

```
docs/engineering/design/
├── 000-template.md
├── README.md
├── overlay-filesystem.md
├── core/
│   ├── configuration-system.md
│   └── content-pipeline.md
└── api/
    └── webhook-handler.md
```

### Generation

Design documents are generated by the **design-doc** skill. The skill uses the template and project context to produce a complete first draft that can then be reviewed and refined.

### Issue Mapping

Each design document maps to a **GitHub issue**. The issue number is recorded in the document header (`Issue: #NNN`) and serves as the canonical tracking point for discussion, approval, and implementation status.
